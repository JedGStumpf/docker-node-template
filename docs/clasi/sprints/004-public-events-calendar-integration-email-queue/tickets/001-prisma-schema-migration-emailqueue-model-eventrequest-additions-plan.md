# Plan: Prisma schema migration — EmailQueue model, EventRequest additions

## Approach

1. Add 5 new fields to the `EventRequest` model in `schema.prisma`:
   - `meetupEventId` String? — Meetup event ID
   - `meetupEventUrl` String? — Meetup event URL
   - `meetupRsvpCount` Int @default(0) — last-synced RSVP count
   - `googleCalendarEventId` String? — Google Calendar event ID
   - `eventCapacity` Int? — max kids for waitlist enforcement

2. Add new `EmailQueue` model with:
   - `id` String @id @default(uuid())
   - `recipient` String
   - `subject` String
   - `textBody` String
   - `htmlBody` String?
   - `replyTo` String?
   - `attachments` String? (JSON-encoded)
   - `status` String @default("pending") — pending/sent/failed/dead
   - `attempts` Int @default(0)
   - `nextRetryAt` DateTime?
   - `lastError` String?
   - `createdAt` DateTime @default(now())
   - Index on (status, nextRetryAt)

3. Run `npx prisma migrate dev --name sprint4-email-queue-eventrequest-additions`

4. Update `prisma.ts` SQLITE_ARRAY_FIELDS — no new array fields needed (all new fields are scalar).

5. Run `server/prisma/sqlite-push.sh` to regenerate SQLite schema.

6. Run `npm run test:server` to verify all existing tests pass.
